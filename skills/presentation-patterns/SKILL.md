# Presentation Patterns

プレゼン資料とビジネスドキュメントの作成を支援するパターン集。

## テンプレート読み取り

ユーザーが .pptx テンプレートを共有した場合、`pptx-cli` または `python-pptx` で構造を抽出する。

### pptx-cli での読み取り

```bash
# レイアウト一覧を取得
pptx layouts list template.pptx

# プレースホルダー情報を取得（JSON形式）
pptx placeholders list template.pptx --format json

# テーマ・配色情報を取得
pptx inspect template.pptx
```

### python-pptx での読み取り

```python
from pptx import Presentation

prs = Presentation('template.pptx')
for layout in prs.slide_layouts:
    print(f"Layout: {layout.name}")
    for ph in layout.placeholders:
        print(f"  Placeholder: idx={ph.placeholder_format.idx}, "
              f"type={ph.placeholder_format.type}, "
              f"name={ph.name}, "
              f"size=({ph.width}, {ph.height})")
```

### 抽出すべき情報

| 情報 | 用途 |
|------|------|
| レイアウト名 | どのレイアウトが使えるか把握 |
| プレースホルダーの位置・サイズ | テキスト量の目安を決める |
| フォント・サイズ | Markdown 出力時の見出しレベル対応 |
| 配色テーマ | 図表生成プロンプトの色指定に使う |
| マスタースライドの要素 | ロゴ位置、フッター形式の把握 |

## ストーリーフレームワーク

### 1. PREP法（汎用・最も使いやすい）

```
P - Point（結論）    : 「〇〇を提案します」
R - Reason（理由）   : 「なぜなら△△だからです」
E - Example（具体例） : 「実際に□□では...」
P - Point（結論再掲） : 「したがって〇〇を進めるべきです」
```

**向いている場面**: 企画承認、提案、意思決定の場

### 2. 課題解決型（企画書・提案書の定番）

```
1. 現状/課題     : 何が問題か（データで裏付け）
2. 原因分析     : なぜ問題が起きているか
3. 解決策       : 何をするか（具体的に）
4. 期待効果     : どうなるか（定量で示す）
5. 実施計画     : いつ・誰が・どうやるか
6. 投資対効果   : コストとリターン
```

**向いている場面**: 新規プロジェクト提案、予算申請

### 3. ピラミッドストラクチャー（経営層向け）

```
         結論
        /    \
     根拠1   根拠2   根拠3
    / | \   / | \   / | \
  詳細 詳細 詳細 ...
```

**特徴**: 結論から先に伝え、根拠を階層的に展開。
経営層は最初の3枚で判断するため、詳細は後半に回す。

### 4. Before/After型（改善提案）

```
1. Before: 現状の課題（痛みを共感させる）
2. After:  理想の状態（ベネフィットを見せる）
3. How:    どうやって実現するか
4. Proof:  実績・データ・事例
```

**向いている場面**: ツール導入提案、プロセス改善

### 5. 報告型（進捗報告・結果報告）

```
1. サマリー（結論を1枚で）
2. 目標 vs 実績
3. 主要な成果
4. 課題・リスク
5. 次のアクション
```

**向いている場面**: 週次/月次報告、プロジェクト完了報告

## スライド設計の原則

### 1スライド = 1メッセージ

```
悪い例: 「売上推移と顧客満足度と今後の計画」（3つのメッセージ）
良い例: 「売上は前年比30%増」（1つのメッセージ）
```

### 見出しで結論を言う

```
悪い例: 「売上データ」
良い例: 「売上は前年比30%増で目標を達成」
```

### テキスト量の目安

| スライドタイプ | 箇条書き数 | 補足 |
|--------------|-----------|------|
| タイトル | 0-1 | サブタイトルのみ |
| キーメッセージ | 1-3 | 大きな数字 + 一言 |
| 詳細説明 | 3-5 | 各項目は1行 |
| 比較表 | 2-4列 | セル内は短く |

### 数字の見せ方

```
悪い例: 「売上は123,456,789円でした」
良い例: 「売上 1.2億円」（大きく表示）

悪い例: 「前年比で約30%程度の増加が見られました」
良い例: 「+30%」（大きく）+ 「前年比売上増」（小さく補足）
```

## エクセル連携パターン

統計やデータ分析が必要な場合のワークフロー：

```
1. データ整理     → Claude Code でCSV/JSON整理
2. 分析・集計     → Python/pandas で集計スクリプト作成
3. グラフ生成     → matplotlib/plotly で画像出力 or Gemini で生成
4. スライド組み込み → Markdown ドラフトにグラフ配置を記述
```

### データからグラフ生成プロンプトを作る

```bash
# データ概要を確認
> このCSVを分析して、スライドに入れるべきグラフを提案して

# グラフの画像生成プロンプトを作る
> /diagram-prompt このデータの推移を示す折れ線グラフ
```

## 関連コマンド

- `/slide-draft` - ヒアリング → 構成設計 → Markdown ドラフト生成
- `/diagram-prompt` - 図表・画像の Gemini プロンプト作成
