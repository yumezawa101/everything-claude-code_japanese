## 指摘事項

### 中程度

1. `skills/continuous-learning-v2/hooks/observe.sh`

新しい `CLAUDE_CODE_ENTRYPOINT` ガードは、既知の非 `cli` 値（`sdk-ts`、`sdk-py`、`sdk-cli`、`mcp`、`remote`）の有限の許可リストを使用しています。

これは前方互換性の穴を残します: 将来の非 `cli` エントリポイント値はすべてフォールスルーしてインタラクティブとして扱われます。これは PR が防止しようとしている自動セッション観察のまさにそのクラスを再導入します。

より安全なルールは:

- `cli` のみを許可
- 他のすべての明示的エントリポイントを自動化として扱う
- 変数が未設定の場合のデフォルトフォールバックを `cli` に保つ

推奨される形:

```bash
case "${CLAUDE_CODE_ENTRYPOINT:-cli}" in
  cli) ;;
  *) exit 0 ;;
esac
```
