# テスト要件

## 最小テストカバレッジ: 80%

テストタイプ（すべて必須）：
1. **ユニットテスト** - 個別の関数、ユーティリティ、コンポーネント
2. **統合テスト** - API エンドポイント、データベース操作
3. **E2E テスト** - クリティカルなユーザーフロー（言語ごとにフレームワークを選択）

## テスト駆動開発

必須ワークフロー：
1. まずテストを書く（RED）
2. テストを実行 - 失敗するはず
3. 最小限の実装を書く（GREEN）
4. テストを実行 - パスするはず
5. リファクタリング（IMPROVE）
6. カバレッジを確認（80%以上）

## テスト失敗のトラブルシューティング

1. **tdd-guide** agent を使用
2. テストの分離を確認
3. モックが正しいか検証
4. テストではなく実装を修正（テストが間違っている場合を除く）

## Agent サポート

- **tdd-guide** - 新機能に積極的に使用、テストファースト を強制
