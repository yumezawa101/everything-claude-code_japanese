# Diagram Prompt - 図表・画像生成プロンプト作成

スライドに入れる図表やビジュアルのための Gemini 画像生成プロンプトを作成する。
画像生成はコストがかかるため、プロンプトを必ず確認してから実行する。

## ワークフロー

### Step 1: 要件の確認

何を図にしたいか確認する：

```
どんな図表が必要ですか？
  a) フロー図（プロセス、画面遷移、業務フロー）
  b) 比較表/マトリクス（競合比較、機能比較）
  c) グラフ（棒/折れ線/円/ファネル）
  d) 概念図（アーキテクチャ、関係図）
  e) アイコン/イラスト（装飾用ビジュアル）
  f) モックアップ（アプリ画面、UI イメージ）

用途は？
  - パワポに貼る（背景白/透過推奨）
  - 資料の表紙画像
  - 概念を伝えるためのイメージ図
```

### Step 2: プロンプト作成

要件に基づいてGeminiプロンプトを作成する。以下の要素を含める：

```
[構図]: 何をどう配置するか
[スタイル]: フラットデザイン / ミニマル / ビジネス / 手描き風 等
[色]: ブランドカラー、配色指定
[テキスト]: 図中に入れる日本語テキスト
[サイズ]: 16:9（スライド向け）/ 1:1 / 4:3
[背景]: 白背景 / 透過 / グラデーション
```

プロンプト例：

```
ビジネス向けのフラットデザインで、ファネル図を作成してください。
上から順に「認知（10,000人）」「興味（3,000人）」「検討（500人）」
「購入（100人）」の4段階。色はグラデーション（青→緑）。
背景は白。16:9のアスペクト比。日本語テキスト。
余計な装飾は不要でクリーンなデザイン。
```

### Step 3: プロンプトの確認（必須）

作成したプロンプトをユーザーに提示し、確認を取る。

```
以下のプロンプトで Gemini に画像生成を依頼します：

---
[プロンプト全文]
---

このプロンプトで問題ないですか？
修正したい箇所があれば指摘してください。

※ 画像生成には Gemini API のコストがかかります。
```

**ユーザーの明示的な承認なしに画像生成を実行しないこと。**

### Step 4: Gemini 実行

ユーザーが承認したら、以下のいずれかの方法で実行する：

**方法A: Gemini MCP が設定済みの場合**
MCP 経由で直接画像生成を実行し、結果をファイルに保存する。

**方法B: MCP 未設定の場合**
プロンプトをそのままコピーして使えるように整形して出力する。
ユーザーが手動で Gemini（Google AI Studio / Gemini アプリ）に貼り付けて使う。

### Step 5: 結果の確認

画像生成結果を確認し、必要に応じてプロンプトを調整する。

```
生成された画像を確認してください。
  a) OK → このまま使う
  b) 微調整 → プロンプトを修正して再生成
  c) やり直し → 別のアプローチで再作成
```

## プロンプトのコツ

### スライド向け画像で効果的な指定

- 「ビジネスプレゼンテーション向け」と明記する
- 「クリーンでミニマルなデザイン」を指定する
- 「白背景」を指定すると PowerPoint に貼りやすい
- 日本語テキストが必要な場合は明示する
- 「16:9」をアスペクト比として指定する

### 避けるべきこと

- 一度に複雑すぎる図を要求しない（シンプルに分割する）
- テキストが多すぎる図は避ける（読めなくなる）
- 写真リアルな画像は避ける（ビジネス資料にはフラットデザインが適切）
