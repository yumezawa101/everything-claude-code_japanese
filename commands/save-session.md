# Save Session コマンド

このセッションで起こったすべて（構築したもの、うまくいったこと、失敗したこと、残っているもの）をキャプチャし、次のセッションが中断した場所から正確に再開できるように日付付きファイルに書き込みます。

## 使用するタイミング

- Claude Code を閉じる前の作業セッション終了時
- コンテキスト制限に達する前（まずこれを実行し、新しいセッションを開始）
- 覚えておきたい複雑な問題を解決した後
- 将来のセッションにコンテキストを引き継ぐ必要がある時

## プロセス

### ステップ 1: コンテキストの収集

ファイルを書く前に以下を収集:
- セッション中に変更されたすべてのファイルを読み取り
- 議論、試行、決定された内容を確認
- 遭遇したエラーとその解決方法（または未解決）を記録
- 関連する場合は現在のテスト/ビルド状態を確認

### ステップ 2: sessions フォルダの作成

```bash
mkdir -p ~/.claude/sessions
```

### ステップ 3: セッションファイルの書き込み

`~/.claude/sessions/YYYY-MM-DD-<short-id>-session.tmp` を作成。

### ステップ 4: すべてのセクションを正直に記入

セクションをスキップしない -- 内容がない場合は "N/A" と記入。不完全なファイルは空のセクションより問題が大きい。

### ステップ 5: ユーザーに表示

書き込み後、内容全体を表示して確認を求める。要求があれば編集を実施。

## セッションファイル形式

```markdown
# Session: YYYY-MM-DD

**Started:** [開始時刻（わかる場合）]
**Last Updated:** [現在時刻]
**Project:** [プロジェクト名またはパス]
**Topic:** [セッションの内容の一行要約]

---

## 構築中のもの
[1-3段落でゴールを説明]

---

## うまくいったもの（証拠付き）
- **[成功項目]** -- 確認方法: [具体的な証拠]

---

## うまくいかなかったもの（理由付き）
- **[試したアプローチ]** -- 失敗理由: [正確な理由/エラーメッセージ]

---

## まだ試していないもの
- [アプローチ/アイデア]

---

## ファイルの現在の状態

| ファイル | ステータス | 備考 |
| ------- | --------- | ---- |
| `path/to/file.ts` | Complete | [内容] |
| `path/to/file.ts` | In Progress | [完了分、残り] |

---

## 行った決定
- **[決定]** -- 理由: [なぜこれを選んだか]

---

## ブロッカーと未解決の質問
- [ブロッカー/未解決の質問]

---

## 次の正確なステップ
[わかる場合: 再開時の最初にすべきこと]
```

---

## 注意事項

- 各セッションは独自のファイルを持つ -- 前のセッションのファイルに追記しない
- 「うまくいかなかったもの」セクションが最も重要 -- これがないと将来のセッションが失敗したアプローチを再試行する
- セッション途中の保存要求があった場合、現在わかっていることを保存し進行中の項目を明確にマーク
- ファイルは `/resume-session` で次のセッション開始時に Claude が読むためのもの
- 正規のグローバルセッションストアを使用: `~/.claude/sessions/`
- 新しいセッションファイルには短縮 ID ファイル名形式を推奨
